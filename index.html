Virtual Payment and State Channels

Non-technical description

Perun network is a system of payment and state channels over Ethereum. Perun's channels can be virtual, which means that the payments over longer chains of channels do not require interaction with the intermediate parties. Highly scalable

Perun is even more scalable compared to other off-chain solutions, since it eliminates the need for all middlemen in a channel network to acknowledge every transaction. This means all transactions within Perun are peer-to-peer without having to bother other parties in the network.

Provable secure

An important feature of Perun is that its protocols have been proven secure using the universal composability framework.

Privacy preserving

Since we eliminate the need for any middleman to acknowledge every transaction routed over them, a middleman can only see the balance of a virtual channel after the closing transaction. This prevents the third party to see when and how often money was exchanged over the virtual channel.

Cost efficient

Perun reduces the cost for executing smart contracts by a manyfold. We keep the transactions off-chain while simultaneously eliminating the fees needed to route transaction over a third party in the channel network.

Lightweight

Perun relies on common cryptographic primitves like cryptographic signature schemes. These are easy to compute, resulting in Perun being executable on almost every device.

Flexible

Perun supports generalized state channels, thus can be used to execute arbitrary smart contract off-chain. Since it's proven secured, it can be used whenever an authority exists. This authority can be a blockchain, a trusted party or even a bank.

Implementation

Our implementation will be developed together with Bosch GmbH. The node software will be published as soon as it is ready. The repository contains a proof-of-concept implementation of Perun's virtual payment channels of length 2.
